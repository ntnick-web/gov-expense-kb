# 情境卡片自動化與 Claude Code 應用建議報告
專案：政府支出法規知識庫（gov-expense-kb）
範圍：情境卡片自動生成、條件問答設計、Claude Code 任務分工
前提條件：Python 已有腳本環境、GitHub Actions 可用、僅使用免費工具 + Claude 訂閱


## 一、整體策略
系統已有 513 個結構化 MD 節點（含 summary、tags、status），是生成情境卡片的最佳原料。
核心策略為：從現有節點推導情境，不從零創作，並依任務性質區分由 Claude Code 或 Python 腳本處理。


## 二、自動化情境卡片生成
### 2.1 做法：規則式生成，不需 LLM
從每個 status: current + review_level: verified 節點的 summary 與 tags，自動產生情境卡片草稿。

for node in nodes:

    if node["status"] != "current": continue

    if node["review_level"] != "verified": continue

    scenario = {

        "id": f"sc_{node['id']}",

        "question": summary_to_question(node["summary"]),  # 規則轉換

        "tags": node["tags"],

        "primary_ids": [node["id"]],

        "related_ids": node["related"],

        "source": "auto"

    }

## 三、條件問答設計：Decision Tree（樹狀選項）
### 3.1 選擇理由
考量系統為純靜態 HTML、無後端，建議採樹狀選項（Decision Tree）：

純 JSON 驅動，不需 LLM runtime
與現有情境卡片架構相容
使用者點選即可，無需打字
每個分支可精確對應條文節點
### 3.2 資料結構（scenarios.json schema 擴充）
{

  "id": "sc_travel_ticket",

  "question": "出差交通費要怎麼報銷？",

  "type": "decision_tree",

  "branches": [

    {

      "label": "國內出差",

      "next": [

        { "label": "搭火車 / 高鐵", "primary_ids": ["TW-D-012"] },

        { "label": "自行開車",       "primary_ids": ["TW-D-015"] },

        { "label": "搭飛機",         "primary_ids": ["TW-D-013"] }

      ]

    },

    {

      "label": "國外出差",

      "next": [

        { "label": "機票艙等",  "primary_ids": ["TW-F-008"] },

        { "label": "當地交通",  "primary_ids": ["TW-F-011"] }

      ]

    }

  ]

}

設計原則：最多 2–3 層，葉節點直接連到條文卡片抽屜。不做開放式對話，維持查詢精準度。


## 四、GitHub Actions 自動化流程
### 4.1 觸發機制
每次 push 02_markdown/ 變更

  → Python 腳本重新掃描 verified + current 節點

  → 產生 scenarios_auto.json（自動部分）

  → 與 scenarios_manual.json（人工維護）合併

  → 輸出 04_web/data/scenarios.json

  → GitHub Pages 自動部署
### 4.2 workflow 設定
# .github/workflows/build-scenarios.yml

on:

  push:

    paths: ['02_markdown/**']

jobs:

  build:

    runs-on: ubuntu-latest

    steps:

      - uses: actions/checkout@v4

      - run: python 05_scripts/build_scenarios.py

      - run: python 05_scripts/03_build_index.py

      - uses: peaceiris/actions-gh-pages@v3

全程免費，每次法規 MD 更新即自動重建情境卡片。

## 六、Claude Code 的角色與限制
### 6.1 重要前提
訂閱（Pro/Max）與 API 是兩套獨立計費系統：

訂閱提供 claude.ai 與 Claude Code CLI 的使用額度
不等於 API credits，無法用於對外即時問答服務
在自動化迴圈中，一個 session 可能在數分鐘內耗盡每日配額
### 6.2 適合用 Claude Code 完成的任務（一次性、人工監督）
以下任務不需跑迴圈，配額消耗可控，品質最高：

① summary 批次重寫 給 Claude Code 02_markdown/ 目錄，指示改寫所有 verified 節點的 summary 為情境句型（40 字以內），直接覆寫 MD 檔，人工逐批審閱。

② 情境卡片草稿生成 給 Claude Code nodes JSON 和 scenarios schema，讓其從 summary 推導問句、對應 primary_ids、自動分類母題，產出 scenarios_auto.json 初版。

③ Decision Tree 結構設計 針對高頻母題，讓 Claude Code 讀取該母題下所有節點，設計 2–3 層樹狀分支，直接產出符合 schema 的 JSON。

④ tags 審查與清理 掃描所有節點，找出 tag 超過 4 個、含虛詞、或與母題名重複的節點，批次提案修改清單，人工確認後執行。
