# ID 編碼規則

> ID 是整個知識庫的骨架,跨檔引用、JSON 索引、URL 路由全部仰賴。**一旦發布勿輕易變更**。

---

## 1. 格式

```
{類別代碼}-{母題}-{三位序號}
```

**範例**
```
A-國內旅費-005     ← 國內旅費第五條
A-國外旅費-007     ← 國外旅費第七條
C-國內旅費-002     ← 國內旅費第 2 則函釋
C-國外旅費-007     ← 國外旅費解釋彙編第七則(條次「第七則」可解析)
D-國內旅費-001     ← 國內旅費 Q&A 第 1 題
B-國外旅費-001     ← 國外旅費生活費標準表
N-政府支出-001     ← 「政府支出」分類節點
```

**「條次」格式**(來自來源 MD 的 metadata)
- 要點/辦法:`第N條`(中文數字),例:`第五條`、`第十五條`
- 點:`第N點`(中文數字),例:`第三點`
- 解釋彙編:`第N則`(中文數字),例:`第七則`;若帶 sub-item(`第四-1則`)cn2int 解析失敗,fallback 為「字典序自動編號」
- Q&A:`Q1`(阿拉伯數字)

---

## 2. 類別代碼

| 代碼 | 類別 | 對應 `type` 欄位 | 說明 |
|------|------|------------------|------|
| `A` | 核心法規 | `核心法規` | 主管機關發布之要點、辦法、規則之條文 |
| `B` | 支出標準 | `支出標準` | 標準表、額度表、費率表 |
| `C` | 解釋函令 | `解釋函令` | 主管機關對特定問題之函釋 |
| `D` | 問答集 | `問答集` | 常見問題與解答 |
| `N` | 分類節點 | `分類節點` | 虛擬節點,僅供關聯圖譜使用 |

**規則**
- 類別代碼為單一英文大寫字母
- 不使用 E、F 等其他字母,需擴充時於本檔登記
- 一個節點僅屬一個類別,不可跨類

---

## 3. 母題清單

母題為知識庫的二級分類,需事先固定,新增需於本檔登記後生效。

### 3.1 核銷規定母題(內環 6 項,對應圖1)

| 母題 | 說明 |
|------|------|
| `國內旅費` | 國內出差旅費報支 |
| `國外旅費` | 國外出差旅費報支 |
| `講座鐘點費` | 講座、授課鐘點費 |
| `酬勞費` | 各類稿費、出席費、審查費 |
| `國外專家` | 邀請國外專家來臺費用 |
| `其他` | 不屬上述者(如電話費、會議費) |

### 3.2 通用/機關專章(外環 3 項)

| 母題 | 說明 |
|------|------|
| `支出憑證與結報` | 政府支出憑證處理要點、經費結報常見疑義問答集等 |
| `教育部專章` | 教育部適用之特別規定 |
| `國科會專章` | 國科會(國家科學及技術委員會)適用之特別規定 |

### 3.3 母題命名規則

- 使用繁體中文
- 2-6 字
- 不含標點符號
- 不含「之」「的」等虛字

### 3.4 新增母題流程

1. 於本檔 §3.1 或 §3.2 表格新增一列
2. 在 `02_markdown/{類別}/` 下建立同名子資料夾
3. 在 `docs/decisions.md` 記錄新增理由
4. 通知 AI 助手重新讀取 `CLAUDE.md`

## 4. 序號規則

- 三位數,從 `001` 開始
- 不重用已刪除節點之序號(避免歷史引用失效)
- 序號區段建議:
  - `001-099`:第一批匯入
  - `100-199`:後續新增
  - `900-999`:測試用,正式環境勿用

### 4.1 各類別序號意涵

| 類別 | 序號意涵 | 實作 |
|------|----------|------|
| `A` 核心法規 | 條次中文數字 → 阿拉伯三位 | `cn2int(meta.條次)`,失敗 fallback 字典序 |
| `B` 支出標準 | 字典序遞增 | `02_parse.py` 的 unassigned 排序邏輯 |
| `C` 解釋函令 | 條次可解析者依條次,否則字典序遞增 | 例:`第七則 → 007`;`第四-1則` cn2int 失敗 → 字典序給後續可用序號 |
| `D` 問答集 | 依 Q&A 題號(Q1 → 001) | regex `Q(\d+)` |
| `N` 分類節點 | 依層級由上到下遞增 | (虛擬,目前未產生) |

**為何 C 類用「字典序」而非「發布日期」**:解釋彙編檔名前綴(`則000一` / `則四-1`)在字典序下天然有序,且可重現(同樣資料每次重建得到同樣 ID);發布日期則需 manifest 完整支援,manifest 缺資料時會破裂。

## 6. 驗證規則

`05_scripts/04_validate.py` 應檢查:

- [x] 格式符合 `[A-N]-[\u4e00-\u9fa5]+-\d{3}` 正則
- [x] 類別代碼為合法值
- [x] 母題在合法清單內
- [x] ID 全域唯一
- [x] 類別與檔案所在資料夾一致(A 類必須在 `A_核心法規/`)
- [x] 母題與所在子資料夾一致

---

## 7. 變更管理

ID 規則一旦發布,**禁止以下變更**:
- ❌ 變更已存在 ID 的格式
- ❌ 移除已使用的類別代碼或母題
- ❌ 重用已刪除節點的序號

**允許變更**:
- ✓ 新增類別代碼(於本檔登記)
- ✓ 新增母題(於本檔登記)
- ✓ 廢止某節點(設 `deprecated: true`,不刪除)

---

## 8. URL 對應(未來上網時)

ID 直接對應 URL slug:

```
https://example.com/kb/A-國內旅費-005
https://example.com/kb/C-國內旅費-002
```

UTF-8 中文 URL 大部分瀏覽器支援,若上線後發現問題再加 URL encode 處理層。
