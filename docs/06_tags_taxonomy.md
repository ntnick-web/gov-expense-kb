# 標籤分類系統(Tags Taxonomy)

> 標籤雙軌制:**母標籤**(分類用,概念規範)+ **自由標籤**(搜尋/關聯用,實際寫入 MD 的 `tags` 欄位)。

---

## 1. 雙軌制說明

### 母標籤(Hierarchical Tags) — **概念規範**
- 預先定義,結構穩定
- 對應「母題」之下的「支出類別」分類概念
- 用於設計參考,實際運作見下方「實作對應」段
- 新增需於本檔登記

### 自由標籤(Free Tags) — **實際 tag**
- MD 的 `tags` 欄位內容
- 可彈性新增,無需事前登記
- 用於跨母題搜尋、橫向關聯
- 過多時定期審視合併

### 實作對應(關鍵)

本檔的「母標籤」是規範層,**實際分類由前端 [04_web/index.html](../04_web/index.html) 內的 `EXPENSE_LAYER` 表決定**。`EXPENSE_LAYER` 各類別有 `match` 詞庫,從節點的 `tags`(自由標籤)反推支出類別,例:

```js
{ name: '交通費', match: ['交通費', '機票', '飛機', '艙等', '計程車', '租車', ...] }
```

母標籤(本檔)是「設計概念」,`EXPENSE_LAYER` 是「實作對應」。本檔母標籤可能比 `EXPENSE_LAYER` 少(設計階段)或不同名(實作做了合併、加了「程序總則」等)。

## 3. 自由標籤命名規則

- 使用名詞,不用動詞(✓ 交通費 / ✗ 報支)
- 2-6 個字
- 跨母題可共用
- 不含標點符號
- 同義詞統一(預先建立對照表見 §5)

## 8. 「程序總則」設計

`EXPENSE_LAYER` 在每個母題的清單末尾(「其他」之前)加一個 **程序總則** 類別,專門兜底「無具體費用 tag」的綜合性條文(如總則、法源、調任、懲處等)。

### match 詞庫(「程序總則」常見命中)
- 泛 tag:`結報核銷`、`國外旅費`(母題自身名)、`國內旅費`
- 程序型 tag:`總則`、`法源依據`、`公差派遣`、`報支期限`
- 行政事項 tag:`調任`、`準用`、`懲處`、`休職`、`撤職`、`停職`、`免職`

### 為何放在 `EXPENSE_LAYER` 結尾
- 命中時優先讓「具體費用類別」(交通費/生活費/...)先吃,只有真的沒具體費用 tag 的節點才落入「程序總則」
- 結果:極少節點落到最後 fallback 的「其他」(目前國內 0 / 國外 0~1)

### 設計理念
不是「費用種類」而是「條文性質」。使用者校對時若補上具體費用 tag,腳本下次重建索引時自動歸正類別,**不需動 `EXPENSE_LAYER` 表**。分類隨資料品質自然提升。

## 9. 新增母題的支出類別流程

新增一個母題的 `EXPENSE_LAYER` 子表時:

1. 觀察該母題實際 tag 分布(可用 `python -c "import json; from collections import Counter; ..."` 統計)
2. 在 [04_web/index.html](../04_web/index.html) `EXPENSE_LAYER['{母題}']` 加陣列,每項含 `name` + `match` 詞庫
3. 順序設計原則:
   - **特殊情境**(如「大陸港澳」「出國進修」)放最前,優先過濾
   - **具體費用類別**居中(交通費 / 生活費 / 手續費 / 保險費 / 行政費 / 禮品交際及雜費 ...)
   - **「程序總則」**放結尾(「其他」之前)兜底泛 tag
   - **「其他」**最末,`match: null`(fallback)
4. 重新整理瀏覽器 → 條文庫展開該母題 → 確認分類分布合理
