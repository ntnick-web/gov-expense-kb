# 處理流程 SOP

> 新增來源檔、修訂條文、發布更新的標準作業流程。

---

## 1. 新增來源檔流程

### 步驟 1:歸檔

將檔案放入 `00_source/` 對應「機關」子目錄:

```
00_source/
├── 01_國科會/      ← 國家科學及技術委員會發布之檔案
├── 02_成功大學/    ← 國立成功大學發布之檔案
├── 03_教育部/      ← 教育部發布之檔案
└── 04_主計總處/    ← 行政院主計總處發布之檔案
```

> 註:類別(A/B/C/D)分流是 `01_extracted/` 之後的事,由 `01_extract.py` 自動依檔頭 metadata 或 manifest 判斷。

**檔名規則**
- 保留原始檔名(便於對照)
- 支援 PDF / DOCX / MD 三種格式
- 若檔名過長,以 `{法規名}_{識別}.ext` 重命名
- 範例:`行政院主計總處_國內旅費報支要點第2點3函釋.pdf`

### 步驟 2:更新清冊(可選)

編輯 `00_source/_manifest.csv`:

```csv
filename,category,parent,agency,version,notes
行政院主計總處_國內旅費報支要點.pdf,A,國內旅費,行政院主計總處,2024-01-15,
行政院主計總處_國內旅費報支要點第2點3函釋.pdf,C,國內旅費,行政院主計總處,2023-08-10,搭配第五條
```

manifest 不是必填:`01_extract.py` 會優先用 manifest,沒對應時從來源 MD 的 `**文件類型:**`/`**來源法規:**` 欄位推斷類別與母題。

### 步驟 3:抽取文字

```bash
python 05_scripts/01_extract.py --file 行政院主計總處_國內旅費報支要點.pdf
```

產物:`01_extracted/{原檔名}.txt`

**檢查點**
- 開啟 `.txt` 確認文字正確抽出
- 中文無亂碼
- 表格類內容是否需要特別處理

若文字抽取失敗(掃描型 PDF),改用 OCR:
```bash
python 05_scripts/01_extract.py --file XXX.pdf --ocr
```

### 步驟 4:結構化切分

```bash
python 05_scripts/02_parse.py
```

產物:`02_markdown/{類別}/{母題}/*.md`(每份 .txt 對應一份 MD,1:1 映射)

腳本會:
- 自動填入 `id`(中文數字 → 阿拉伯三位)、`type`、`parent`、`source`、`version`
- 抽出短描述當 title 後綴(失敗則 `_TODO`)
- 從 metadata 標籤拆出 `tags`(過濾與 parent/type 同名的泛 tag)
- 處理 Q&A 格式為「## 問題 / ## 回答」雙區塊
- **不寫入 `reviewed`**(留給步驟 5 / 5b)
- 重點摘要、相關規定區塊填 `(待人工補)` placeholder

### 步驟 5a:批次自動初校(推薦,可省大量人工)

```bash
python 05_scripts/_batch_autoreview.py
```

對所有沒 `reviewed` 的 MD:
- 抽 `## 條文全文 / 函釋全文 / 標準全文 / 問題` 區塊首段(到第一個「。」)為「重點摘要」
- 加 `reviewed: 今日`、`version: TODO → 2024-01-01` placeholder
- 摘要尾標 `_(自動初校,待人工潤飾)_` 斜體

執行後所有節點都進入「可瀏覽」狀態,前端不再顯示「草稿」flag。

### 步驟 5b:人工精校(挑重要節點做)

開啟重要 MD(尤其使用率高的條文),逐份補:

**必補欄位**
- `tags`:依條文內容補 2-5 個更精準的標籤(取代/精化自動標籤)
- `related`:其他相關條文/函釋/Q&A 的 ID(讓關聯圖更豐富)

**潤飾**
- 重點摘要:刪掉自動產的內容,改寫為 3-5 句重點(刪掉 `_(自動初校,待人工潤飾)_` 標記)
- 相關規定:補連結 + 一兩句補充說明
- 備註:版本變更、實務注意事項

**必核對**
- 條文全文是否完整
- 標題是否正確
- 條文編號是否正確

> 人工精校不需一次做完;`_batch_autoreview.py` 已讓所有節點可用,可以挑使用率高的逐步精校。

### 步驟 6:重建索引

```bash
python 05_scripts/03_build_index.py
```

產物:`03_index/nodes.json`、`edges.json`、`tags.json`、`search_index.json`

### 步驟 7:驗證

```bash
python 05_scripts/04_validate.py
```

通過則無輸出;有問題則列印錯誤清單。常見錯誤:
- ID 重複或格式錯誤
- `related` 引用不存在的 ID
- `parent` 不在合法清單
- `version` 日期格式錯誤

### 步驟 8:視覺驗證

啟動本機 server 後開瀏覽器:

```bash
python -m http.server 8765
# → 開瀏覽器訪問 http://localhost:8765/04_web/
```

或用 Claude Code:`preview_start "Static frontend (04_web)"`(從 [.claude/launch.json](../.claude/launch.json) 啟動)。

> ⚠️ 不能直接 `open 04_web/index.html`,因為 file:// 協定下瀏覽器會擋 `fetch()`,JSON 與 MD 載不進來。

檢查項:
- 泡泡圖中新增母題的泡泡大小有反映節點數變化
- 條文庫分類樹三層結構正確,展開支出類別後看到核心法規/解釋函令/問答集
- 點任一卡片,抽屜載入 MD 正確
- 搜尋可命中(Ctrl+K)
- 關聯圖節點與推斷邊都顯示
- 從泡泡圖點母題 → 條文庫鎖定該母題 → 切到關聯圖 → scope banner 出現

### 步驟 9:Git 提交

```bash
git add .
git commit -m "新增:國內旅費要點第五條交通費上限及相關函釋"
```

**Commit 訊息規範**
- 動詞開頭:`新增`、`修訂`、`刪除`、`重構`
- 說明範圍:涉及哪個母題、哪個條文
- 一次只做一件事(不混合新增與重構)

## 2. 修訂既有條文流程

### 情境:法規修訂、函釋更新

1. 將新版檔案放入 `00_source/`(舊版保留,加 `_v{舊版本日}` 後綴)
2. 更新 `_manifest.csv`(新增一列,舊列保留並於 `notes` 註記「舊版」)
3. 開啟對應 MD,**直接修改條文全文**
4. 更新 `version` 為新版本日
5. 在「備註」區塊記錄變更摘要
6. 重建索引、驗證、提交

**保留歷史的方法**
- Git 提供完整歷史
- 不另建「舊版本」資料夾
- 重大變更於 `docs/decisions.md` 記錄

---

## 3. 廢止節點流程

### 情境:條文刪除、函釋停止適用

**勿直接刪除 MD 檔**(會破壞既有引用)。

**正確做法**
1. 在 MD front-matter 加 `deprecated: true`
2. 加 `deprecated_reason: "依XXX函釋停止適用"`
3. 加 `deprecated_date: 2025-XX-XX`
4. 重建索引
5. 前端會自動以淡色顯示並標示「已廢止」

---

## 4. 標籤管理流程

### 4.1 新增標籤

- 直接在 MD `tags` 欄位寫入即可
- 重建索引時自動納入 `tags.json`

### 4.2 標籤合併

發現有「交通費」與「交通費用」重複時:
1. 全域搜尋取代:`tags: 交通費用` → `tags: 交通費`
2. 重建索引
3. 於 `docs/06_tags_taxonomy.md` 記錄

### 4.3 標籤分類

母標籤需於 `docs/06_tags_taxonomy.md` 預先定義。
自由標籤可隨意新增,定期審視合併。

## 5. 發布流程(未來上線時)

```bash
# 1. 確保所有變更已提交
git status

# 2. 標記版本
git tag -a v1.0.0 -m "首次正式發布"

# 3. 部署靜態檔
rsync -av 03_index/ 04_web/data/
rsync -av 04_web/ user@server:/var/www/kb/

# 4. 推送 tag
git push origin v1.0.0
```

---

## 6. 緊急修補流程

### 情境:發現條文內容錯誤、函釋誤植

1. **立刻**修改 MD
2. 在備註區加上錯誤說明與更正日期
3. 重建索引
4. Git commit 訊息以 `修正:` 開頭
5. 若已部署,立即重新部署
6. 通知相關使用者

---

## 7. 月度檢查清單

每月初執行一次:

- [ ] 主計總處網站是否有新函釋公告
- [ ] 既有條文是否有版本更新
- [ ] 執行 `04_validate.py` 確認無潛在問題
- [ ] 檢視 `_manifest.csv` 是否與 `00_source/` 一致
- [ ] 備份整個專案資料夾(Git push + 雲端備份)

---

## 8. 故障排除

| 問題 | 可能原因 | 解法 |
|------|----------|------|
| 抽取文字亂碼 | 掃描型 PDF 或編碼問題 | 改用 OCR 模式 |
| 切分結果條次錯亂 | 原文格式特殊 | 手動切分,於 02_parse.py 加例外處理 |
| 索引建立失敗 | 某 MD front-matter 語法錯誤 | 執行 04_validate 找出問題檔 |
| 前端搜尋無結果 | 索引未重建 | 重跑 03_build_index.py |
| 抽屜內容空白 | MD 路徑不對 | 檢查 nodes.json 之 file_path |
